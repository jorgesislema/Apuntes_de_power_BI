# Guía de Contribución

## Bienvenido a Apuntes de Power BI

Gracias por tu interés en contribuir a este proyecto de documentación sobre Power BI. Esta guía te ayudará a entender cómo puedes participar y mejorar estos apuntes.

## Cómo Contribuir

### Reportar Errores o Sugerir Mejoras
- Abre un issue describiendo el problema o la mejora propuesta
- Incluye ejemplos específicos cuando sea posible
- Etiqueta apropiadamente el issue

### Contribuir con Contenido
1. Fork del repositorio
2. Crea una rama para tu contribución
3. Realiza los cambios siguiendo las convenciones establecidas
4. Envía un pull request

## Estándares de Documentación

### Estructura de Archivos
- Usa nombres descriptivos en español
- Mantén la numeración secuencial
- Incluye siempre un glosario por sección

### Formato de Contenido
- Usa Markdown estándar
- Incluye ejemplos prácticos
- Agrega capturas de pantalla cuando sea necesario
- Mantén un tono profesional y educativo

### Convenciones de Escritura
- Usa español técnico preciso
- Explica conceptos complejos paso a paso
- Incluye referencias a documentación oficial
- Agrega casos de uso reales

## Revisión de Contenido

Todo el contenido será revisado para asegurar:
- Precisión técnica
- Claridad en la explicación
- Coherencia con el resto de la documentación
- Actualización con las últimas versiones de Power BI

## Código de Conducta

Mantén un ambiente respetuoso y profesional en todas las interacciones.

¡Gracias por contribuir al conocimiento de la comunidad de Power BI!
